写了个 TodoList

发表于 2018-09-22 | 分类于 JAVA | 阅读次数:
字数统计: 4,582 | 阅读时长 ≈ 24 分钟

因为最近的事情比较多，不仅仅是毕业设计的事，还有秋招的事情也都是扎堆在一起了，每天起床，都有点迷糊。
偶然想到了很久看的关于 Laravel 之父 Taylor Otwell 的一篇文章，我是如何工作的。碰上现在每天开电脑的第一件事就是登录邮箱了。收到邮件，有时候会一直卡在某个点上，为此耗费了太多的时间了。诸如阮一峰老师的《未来世界的幸存者》 一书中写道：在早晨浪费一小时，你得花一整天来弥补。附上原文地址为什么起床后不能收邮件？。反过来思考，如果起床后收到的信息都是自己昨天未完成的待办事项呢？是不是对我们一天时间的规划有所帮助呢？想到这，那就试试看吧。虽然市面上的 todolist 类的产品很多，但我还是选择了自己手撸一个，尽管途中有遇到一些困难，但走过了，也就成长了。

工具

这里我使用了市面上比较流行的两个框架，一个是 Spring Boot，一个是 Vue.js 。刚好 Vue.js 的官方文档，就有一个 TodoMVC，看懂文档，再搭配环境，就可以敲出来了，不过最后打包的时候，还是会遇到一些问题。后端使用的是 Spring Boot，这个真的比Spring 的配置大大简化了好多，并搭配 Maven 管理插件扩展，简直是太舒服了。

打包

打包就容易点了，将前台的文件打包之后 ，只需要将 dist 文件夹下的文件，全部放到 src/main/resources/static 文件夹下面，然后maven 执行 clean package 就好了

最后运行 target/todolist-0.0.1-SNAPSHOT.jar 即可

优化

这样打包之后的jar包其实有几十M，不适合迭代化，所以可以压缩成 Fat jar，方便传输和迭代，不过笔者这里不迭代化，一个jar包，方便管理，所以就没压缩。
