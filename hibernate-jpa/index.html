Hibernate 之 JPA 用法

发表于 2018-09-06 | 分类于 JAVA | 阅读次数:
字数统计: 3,730 | 阅读时长 ≈ 19 分钟
JPA Java Persistence API是Java EE 5的标准ORM接口，也是ejb3规范的一部分。所以就了解一下JPA的一些相关注解，以及Hibernate框架使用JPA进行数据操作！

Hibernate 框架

Hibernate是一种ORM框架，全称为 Object_Relative DateBase-Mapping，在Java对象与关系数据库之间建立某种映射，以实现直接存取Java对象！

JPA Java Persistence API

JPA Java Persistence API是Java EE 5的标准ORM接口，也是ejb3规范的一部分。用于对象持久化的API。以及使得应用程序以统一的方式访问持久层。

JPA和Hibernate之间的关系

JPA 是 hibernate 的一个抽象（就像JDBC和JDBC驱动的关系）：

- JPA 是规范：JPA 本质上就是一种  ORM 规范，不是ORM 框架 —— 因为 JPA 并未提供 ORM 实现，它只是制订了一些规范，提供了一些编程的 API 接口，但具体实现则由 ORM 厂商提供实现
- Hibernate 是实现：Hibernate 除了作为 ORM 框架之外，它也是一种 JPA 实现
从功能上来说， JPA 是 Hibernate 功能的一个子集

JPA 注解

JAP的注解包括有：

  • @Entity
  • @Table
  • @Id
  • @GeneratedValue
  • @Column
  • @Basic
  • @Transient
  • @Temporal

@Entity

@Entity 标注用于实体类声明语句之前，指出该Java 类为实体类，将映射到指定的数据库表

@Table

当实体类与其映射的数据库表名不同名时需要使用 @Table 标注说明。该标注与 @Entity 标注并列使用，置于实体类声明语句之前，可写于单独语句行，也可与声明语句同行。

  1. @Table 标注的常用选项是 name，用于指明数据库的表名
  2. @Table标注还有一个两个选项 catalog 和 schema 用于设置表所属的数据库目录或模式，通常为数据库名。uniqueConstraints 选项用于设置约束条件，通常不须设置。

@Id

  1. 用于声明一个实体类的属性映射为数据库的主键列。该属性通常置于属性声明语句之前，可与声明语句同行，也可写在单独行上。
  2. @Id标注也可置于属性的getter方法之前。

@GeneratedValue

用于标注主键的生成策略，通过 strategy 属性指定。默认情况下，JPA 自动选择一个最适合底层数据库的主键生成策略：SqlServer 对应 identity，MySQL 对应 auto increment。
在 javax.persistence.GenerationType 中定义了以下几种可供选择的策略：

- IDENTITY：采用数据库 ID自增长的方式来自增主键字段，Oracle 不支持这种方式；
- AUTO： JPA自动选择合适的策略，是默认选项；
- SEQUENCE：通过序列产生主键，通过 @SequenceGenerator 注解指定序列名，MySql 不支持这种方式
- TABLE：通过表产生主键，框架借由表模拟序列产生主键，使用该策略可以使应用更易于数据库移植。

@Basic

表示一个简单的属性到数据库表的字段的映射,对于没有任何标注的 getXxxx() 方法,默认即为@Basic

  1. fetch: 表示该属性的读取策略,有 EAGER 和 LAZY 两种,分别表示主支抓取和延迟加载,默认为 EAGER.
  2. optional:表示该属性是否允许为null, 默认为true

@Column

当实体的属性与其映射的数据库表的列不同名时需要使用@Column 标注说明，该属性通常置于实体的属性声明语句之前，还可与 @Id 标注一起使用。

  1. @Column 标注的常用属性是 name，用于设置映射数据库表的列名。此外，该标注还包含其它多个属性，如：unique 、nullable、length等。
  2. @Column 标注的 columnDefinition 属性: 表示该字段在数据库中的实际类型
  3. @Column标注也可置于属性的getter方法之前

@Transient

表示该属性并非一个到数据库表的字段的映射,ORM框架将忽略该属性.
如果一个属性并非数据库表的字段映射,就务必将其标示为@Transient,否则,ORM框架默认其注解为@Basic

@Temporal

在核心的 Java API 中并没有定义 Date 类型的精度(temporal precision). 而在数据库中,表示 Date 类型的数据有 DATE, TIME, 和 TIMESTAMP 三种精度(即单纯的日期,时间,或者两者 兼备).在进行属性映射时可使用@Temporal注解来调整精度.

一对多(双向)

这里我们假设有 客户 和 联系人 两个表，一个客户可以有多个联系人，以此营造一种 一对多的对应关系。

数据库
CstLinkman 实体类

CstLinkman 实体类，getCustomer() 方法使用了注解

  • @ManyToOne 标识多对一的关系，targetEntity 指明对应的实体类，默认可以不写，cascade表示级联关系 ,可指定为ALL,PERSIST,MERGE,REFRESH和REMOVE 等
  • @JoinColumn 表示外键约束，name对应数据表的列名，referencedColumnName对应引用的数据表的列名
CstCustomer 实体类

CstCustomer 实体类中，getLinkmans() 方法使用注解

  • mappedBy:放弃关系的维护，把关系的维护交给多方对象的属性去维护关系
  • fetch: 表示该属性的读取策略,有 EAGER(急加载) 和 LAZY(懒加载)

多对多

这里我们假设有 用户表和角色表，一个用户有多个角色，一个角色有多个用户，以此营造多对多的对应关系

数据库
SysRole 类

这里的 getUsers() 方法 使用ManyToMany 注解

  • @ManyToMany 表示多对多的对应关系
  • @JoinTable 表示关联的表，name为关联的表名
  • @joinColumns 属性表示，在保存关系中的表中，所保存关联关系的外键的字段。并配合 @JoinColumn 标记使用。
  • @inverseJoinColumns 属性与 @joinColumns 属性类似，它保存的是保存关系的另一个外键字段。
